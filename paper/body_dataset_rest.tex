\subsection{Data collection protocol}

All trajectories in Baihu are collected through human teleoperation. The dataset does not include scripted robot rollouts, simulation trajectories, or internet-sourced robot videos. Each episode records a human operator controlling a physical robot platform to complete a requested manipulation task under a specified scene setup.

Task annotations are provided manually by the task requester when the collection requirement is proposed. The task label or instruction therefore reflects the intended task specification before data collection, rather than being generated automatically from the robot logs after collection. The task description defines the expected scene setup, objects or tools, target state, and manipulation procedure for each collection requirement.

Because Baihu spans 13 robot platforms, the observation and action spaces are platform-specific. Camera count, camera viewpoint, camera model, and sensor configuration depend on the robot platform and collection setup. Similarly, action representations are not forced into a single universal schema. Different platforms may use different combinations of arm joints, end-effector commands, gripper or hand commands, and other platform-specific control dimensions. Baihu therefore preserves platform-level schema information while converting the data into a standardized training format.

\subsection{Skill taxonomy}

Baihu covers 64 manipulation skills across 2989 tasks. Rather than treating tasks as isolated labels, the dataset organizes manipulation behaviors through a reusable set of skill-level action semantics. These skills describe both low-level motion primitives and higher-level manipulation interactions, including object acquisition, placement, handover, articulated object interaction, contact-based motion, tool and material manipulation, and navigation-related actions.

These scenario categories provide a practical organization for analyzing how robot policies transfer across environments, object types, and manipulation contexts. At the task level, Baihu contains 2989 tasks, including both short-horizon primitive behaviors and longer-horizon task chains. Many tasks can be viewed as compositions of lower-level manipulation skills, such as grasping, placing, pushing, pulling, handing over, inserting, pressing, opening, closing, sorting, and arranging. This composition makes the dataset useful not only for direct action prediction, but also for studying how low-level manipulation skills combine into longer task sequences.

\subsection{Object and interaction diversity}

Baihu contains robot interactions with diverse real-world objects. The objects span household items, kitchen utensils, retail goods, packages, industrial parts, tools, flexible materials, and irregularly shaped objects. These objects differ in size, shape, material, rigidity, surface texture, and manipulation affordance. Such object diversity is important for learning policies that do not overfit to a small set of familiar objects.

The dataset also includes different interaction patterns between robots and objects. Some trajectories involve single-object pick-and-place behaviors, while others involve object sorting, tool use, assembly-like manipulation, container interaction, drawer or door operation, and multi-object organization. This diversity supports the study of generalization across objects, tasks, and physical interaction types.

\subsection{Temporal diversity}

Baihu contains tasks spanning a broad range of temporal durations. For each of the 2989 tasks, we compute the mean trajectory duration over all trajectories associated with that task and group tasks according to this value. As shown in Figure~\ref{fig:task-duration}, 40.55\% of tasks have a mean trajectory duration below 60 seconds, 74.94\% are below 180 seconds, and 92.17\% are below 300 seconds. Meanwhile, 7.83\% of tasks exceed 300 seconds, including 1.27\% with mean durations of at least 600 seconds. The median task lies within the 60--90 second interval.

\subsection{Data format}

Baihu is converted from HDF5 source records into the LeRobot v2.1 format. Each episode is represented as a trajectory consisting of temporally ordered observation-action pairs. Depending on the robot platform and task setup, each trajectory may include platform-specific visual observations, robot state information, action commands, task labels, and metadata. The standardized format is intended to support direct loading by robot learning pipelines and large-scale model training systems while retaining the platform-specific observation and action schema required by each robot.

A typical Baihu trajectory can be represented as a task instruction or task label, synchronized visual observations, robot proprioceptive states, robot actions, episode metadata, and frame-level timestamps or ordering information.
